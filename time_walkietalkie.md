# 시간 무전 (Radio to Future Self) — 기획 문서

> 미래의 나에게 무전을 보내고, 목표 날짜가 되면 회고하는 자기계발 도구.
> 사물(무전기·엘리베이터·신호등)의 상태 전환 구조를 시간·성찰이라는 새 맥락에 옮겨 담은 프로토타입.

---

## 1. 한 줄 정의

설정한 목표에 따라, 현재의 나가 봉인된 무전을 계속 보내고(적립), 목표 날짜에 도달하면 미래의 나가 열어 회고하는(회고) 순회형 상태 기계 기반 도구.

---

## 2. 배경 / 문제

- 이루고 싶은 목표가 있지만 길이 멀게 느껴지고, 지금의 고민이 언젠가 어떻게 풀릴지 알 수 없다.
- 의도를 적는 일기·자기계발은 *반복*되는 프로세스지만, 과거에 쓴 다짐과 미래의 회고가 서로 연결되지 않고 흩어진다.
- "미래의 나"에게 지금을 봉인해 보내고, 그날이 왔을 때 진짜로 열어보는 구조로 이 반복에 시간의 축을 부여한다.

---

## 3. 설계 영감 — 세 사물의 상태 전환 구조

이 프로토타입은 세 사물의 공통 구조("정해진 상태들을 조건에 따라 순회하는 상태 기계")에서 출발했다.

| 사물 | 기여한 구조 | 프로토타입 반영 |
|---|---|---|
| 무전기 | 반이중 — 동시에 만날 수 없고, 한쪽이 보내면 다른 쪽이 받음 | 현재의 나(송신) ↔ 미래의 나(수신)가 시간으로 분리 |
| 엘리베이터 | 목표 도달 — 목표를 정하고 향하다 도달하면 국면 전환 | 목표 날짜로 향하다 도달 시 적립→회고 전환 |
| 신호등 | 자동 전환·상태 직결 — 시간이 되면 저절로 바뀌고 색으로 드러남 | 목표 날짜 도달 시 자동으로 회고 전환, 국면을 색으로 표시 |

---

## 4. 핵심 구조 — 순회형 상태 기계 (시간 전환형)

### 국면(Phase) — 시간에 의해 자동 전환

```
[적립 국면]  현재의 나가 봉인 메시지를 계속 송신 (reinforce 반복)
     │
     │ 목표 날짜 도달 (시간 트리거 = 자동 전환)
     ▼
[회고 국면]  미래의 나가 봉인을 열어 그때의 나에게 응답
```

- 워키토키의 *수동 왕복*이 아니라, 신호등처럼 *시간에 의한 자동 전환*이다.
- 목표 도달은 종료가 아니라 국면 전환이다. 회고 후 새 목표로 사이클이 반복될 수 있다(reuse).

### 상태 = 변환 스위치

같은 "녹음" 행위라도 현재 국면(phase)에 따라 다른 의미로 변환된다.
- 적립 국면의 녹음 → "미래로 봉인해 보내기"
- 회고 국면의 녹음 → "그때의 나에게 답하기"

---

## 5. 두 반복 구조 (프로세스 CT)

이 도구는 두 층위의 반복을 가진다. 성격이 다르므로 구현 시 구분한다.

### 5-1. reinforce — 적립 국면 내부 반복 (강화)

- **무엇이 도나**: 봉인 메시지 (매번 같은 형식: 의도·고민·다짐)
- **방향**: 축적 — `thread[]`에 계속 push
- **데이터 성격**: 날것 그대로 쌓임 (양적 증가)
- **의미**: 같은 목표를 향해 반복적으로 다짐을 강화

### 5-2. reuse — 사이클 전체 반복 (재사용)

- **무엇이 도나**: 회고에서 추출한 배움·통찰 (가공됨)
- **방향**: 변환 후 이월 — 배움 → 다음 목표의 재료
- **데이터 성격**: 압축·정제됨 (질적 변화)
- **의미**: 한 목표의 회고 결과를 다음 목표 설정에 재사용

> 핵심 대비: reinforce는 *날것을 쌓기*, reuse는 *압축해서 넘기기*.

---

## 6. 데이터 흐름

| 단계 | 입력 데이터 | 처리 | 출력(다음으로 넘김) |
|---|---|---|---|
| 목표 설정 | 목표, 날짜, 기간 | 봉인 기간 설정 | 목표·기간 → 적립 |
| 적립 (reinforce) | 지금의 의도·고민 | 녹음 → 봉인 | 봉인 메시지 → `thread[]` 축적 |
| 회고 | 봉인 묶음 + 시간 도달 | 개봉 → 응답 | 회고 응답 → 배움 |
| 배움 추출 | 회고 응답 | 통찰로 압축 | 배움 → 새 목표 (reuse) |

---

## 7. 변수 · 상수 · 상태 (시스템 정의)

### 상수 (Constants) — 한 번 정하면 세션 내 불변

| 이름 | 의미 | 타입/예시 |
|---|---|---|
| `goal` | 목표 (도달한 미래의 나) | string, "포트폴리오를 완성한 나" |
| `targetDate` | 목표 날짜 | date |
| `totalDays` | 목표까지 전체 기간(일) | int, 30/90/180/365 |
| `MAX_MEMO_SEC` | 메모 최대 길이 | int, 60 |

### 변수 (Variables) — 쓰는 동안 변함

| 이름 | 의미 | 타입 |
|---|---|---|
| `phase` | 국면 (0=적립, 1=회고) | int (0/1) |
| `daysLeft` | 남은 날 | int |
| `progress` | 진행률 (%) | int, (totalDays−daysLeft)/totalDays×100 |
| `isRecording` | 녹음 중 여부 (반이중 잠금) | bool |
| `memoLength` | 현재 녹음 길이(초) | int |
| `thread[]` | 현재의 나가 보낸 봉인 메시지 배열 | array |
| `replies{}` | 회고에서 미래의 나가 답한 것 | map (memoId→reply) |
| `reachedGoal` | 목표 도달 여부 | bool |

### 상태 전이 규칙 (if-then)

```
IF isRecording == true          → 다른 입력 잠금 (반이중)
IF phase == 0 AND 전송          → thread.push(봉인메시지), daysLeft 감소
IF daysLeft <= 0                → phase = 1 (회고 자동 전환), reachedGoal = true
IF phase == 1 AND 개봉          → 해당 메시지 열림
IF phase == 1 AND 응답          → replies[memoId] = 회고응답
IF memoLength > MAX_MEMO_SEC    → 녹음 자동 정지·전송
```

> 상태 충돌 점검: `phase`는 0/1 배타적, `isRecording` 잠금으로 동시 입력 방지 → 모든 상태에서 조건 충돌 없음.

## 8. 화면 구조

### 화면 1 — 목표 설정 (Setup)
- 입력: 목표 텍스트 ("어떤 목표에 도달한 나인가?")
- 선택: 기간 칩 (30일 / 90일 / 180일 / 1년)
- 버튼: "무전 시작하기" → 적립 국면 진입

### 화면 2 — 적립 국면 (Accrue)
- 테마: 따뜻한 색 (현재의 나)
- 목표 + 진행바 (D-day, %)
- 국면 표시: "적립 중 · 현재의 나"
- 봉인 메시지 목록: 🔒 Day N · 봉인된 메시지 (내용 비공개, 길이만 표시)
- 녹음 컨트롤: "현재의 나로 녹음" → 파형·초 카운트 → "미래로 봉인해 보내기"
- (데모) "목표 날짜로 가기 → 회고 시작" 버튼 — 시간 압축

### 화면 3 — 회고 국면 (Review)
- 테마: 서늘한 색 (미래의 나) — 자동 전환
- 국면 표시: "회고 중 · 미래의 나"
- 봉인 목록: "Day N의 나 · 열어보기" → 개봉
- 개봉 카드: 과거의 나 메시지(재생) + "미래의 나로 회고하기"(녹음 응답)

---

## 9. 인터랙션 원칙

- **반이중**: 한 번에 하나만 녹음. 녹음 중엔 다른 조작 잠금.
- **봉인**: 적립 중엔 보낸 메시지 내용을 다시 볼 수 없음 (진짜 시간차 구현).
- **자동 전환**: 목표 날짜 도달 시 사용자 조작 없이 회고로 전환.
- **상태 직결**: 국면을 색으로 즉각 지각 (적립=따뜻, 회고=서늘).
- **순회**: 목표 도달해도 종료되지 않음. 회고 후 새 목표로 재사용 가능.

---

## 10. 비주얼 방향

- 무전기의 물성(파형, 송수신) + 시간의 거리감(현재=가깝고 따뜻, 미래=멀고 서늘)
- 현재의 나: 따뜻한 톤 (예: 주황 계열)
- 미래의 나: 서늘한 톤 (예: 청록 계열)
- 음성 메모 UI: 파형 애니메이션, 재생 버튼, 길이(초) 표시
- 흔한 템플릿 룩(크림+세리프) 회피, 상태를 색으로 직결

---

## 11. 데모 시나리오

1. 목표 설정: "포트폴리오를 완성한 나", 90일 선택
2. 적립: "현재의 나로 녹음" → 고민 녹음 → "미래로 봉인해 보내기" (2~3회 반복, D-day 감소)
3. 시간 압축: "(데모) 목표 날짜로 가기" 클릭
4. 자동 전환: 회고 국면으로 (테마 색 변화)
5. 회고: "Day N의 나 · 열어보기" → 과거 메시지 재생 → "미래의 나로 회고하기" 응답
6. (확장) 회고 배움 → 새 목표 설정으로 사이클 반복

---

## 12. 확장 아이디어 (선택)

- **reuse 명시화**: 회고 후 "이번에 배운 것"을 추출해 다음 목표 설정 화면에 이월
- **알림**: 목표 날짜 도달 시 실제 푸시 (봉인 개봉 유도)
- **실제 음성 녹음**: 목업 파형을 실제 오디오 녹음/재생으로 대체
- **적립 리듬**: 정기 알림으로 reinforce 반복 유도 (일기 습관화)
